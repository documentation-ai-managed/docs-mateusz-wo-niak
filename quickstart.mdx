## Prerequisites

Before you begin, ensure you have:

- A modern web browser like Chrome, Firefox, or Safari
- An email address for account verification
- Optional: A GitHub account for version control integration

<Callout kind="info">

Prepare your project details, such as name and description, to streamline setup.

</Callout>

## Create Your Account

Follow these steps to sign up and log in to Mateusz Wo≈∫niak's documentation platform.

<Steps>
  <Step title="Visit the Sign-Up Page" icon="globe">
    Navigate to `https://docs.example.com/signup` in your browser.
  </Step>
  <Step title="Enter Your Details" icon="user">
    Provide your email, create a strong password, and complete the CAPTCHA.
  </Step>
  <Step title="Verify Email" icon="mail">
    Check your inbox for a verification link and click it to activate your account.
  </Step>
  <Step title="Log In" icon="log-in">
    Return to `https://docs.example.com/login` and sign in with your credentials.
  </Step>
</Steps>

## Create a New Documentation Space

Once logged in, set up your dedicated space for project docs.

<Steps>
  <Step title="Access Dashboard" icon="layout-dashboard">
    From the homepage, click the `New Space` button in the top-right corner.
  </Step>
  <Step title="Configure Space" icon="settings">
    Enter a name like `My Project Docs`, add a description, and select visibility (public or private).
  </Step>
  <Step title="Customize Branding" icon="palette">
    Set your brand color to `#3B82F6` and upload a logo if desired.
  </Step>
  <Step title="Save and Enter" icon="save">
    Click `Create Space` to generate your new documentation environment.
  </Step>
</Steps>

## Add Your First Page

Populate your space with content using Markdown or MDX pages.

<Tabs>
  <Tab title="Markdown Page" icon="file-text">
    Create a simple Markdown file for quick notes.

    ````markdown
    # Welcome to My Docs

    This is your first **Markdown** page.

    - List item 1
    - List item 2

    ```
    code block
    ```
    ````

  </Tab>
  <Tab title="MDX Page" icon="file">
    Use MDX for interactive components.

    ````mdx
    ## Interactive Content

    <Callout kind="tip">MDX supports components!</Callout>

    ```javascript
    console.log("Hello, Docs!");
    ```
    ````

  </Tab>
</Tabs>

<Callout kind="tip">

Save your page with `Cmd+S` (Mac) or `Ctrl+S` (Windows/Linux) to publish instantly.

</Callout>
